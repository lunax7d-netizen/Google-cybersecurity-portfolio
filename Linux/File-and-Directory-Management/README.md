# Linux File and Directory Management

## Objective

Practice managing files and directories in a Linux environment by creating, removing, moving, and editing files using Bash commands.

## Scenario

As part of a cybersecurity lab, I practiced organizing a Linux directory structure by creating new directories, removing unnecessary files, moving files between locations, and editing documentation files. These tasks represent common activities performed by security analysts when managing system data.

## Tools Used

- Linux
- Bash
- Nano Text Editor

## Commands Used

### Create a Directory

```bash
mkdir logs
```

### Remove a Directory

```bash
rmdir temp
```

### Move a File

```bash
mv Q3patches.txt /home/analyst/reports
```

### Remove a File

```bash
rm tempnotes.txt
```

### Create a New File

```bash
touch tasks.txt
```

### Edit a File Using Nano

```bash
nano tasks.txt
```

### View File Contents

```bash
cat tasks.txt
```

### Clear Terminal

```bash
clear
```

## Skills Demonstrated

- Linux File Management
- Directory Organization
- Bash Command Line Usage
- File Creation and Removal
- File Editing with Nano
- Data Organization

## Key Takeaways

- Used `mkdir` to create new directories.
- Used `rmdir` and `rm` to remove unwanted directories and files.
- Used `mv` to move files between directories.
- Used `touch` to create new files.
- Used the `nano` editor to modify file contents.
- Used `cat` to verify file information.

## Outcome

Successfully managed a Linux directory structure by creating, removing, moving, and editing files using Bash commands. This lab strengthened my understanding of Linux file management skills used in cybersecurity operations.
